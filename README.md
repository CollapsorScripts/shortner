## Настройка
В папке проекта есть конфигурационные файлы:
**.env** - содержит настройки переменных окружения для контейнеров **Docker** и самих сервисов
```
DB_USER=имя пользователя БД
DB_PASSWORD=пароль БД
DB_NAME=имя базы данных
DB_PORT=порт базы данных
DB_HOST=хост базы данных (или имя контейнера)
SERVER_PORT=порт который прослушивает api gateway
```

**prod.yaml** - внутренние настройки сервера (расположение ```config/prod.yaml```)
```yaml
env: prod #окружение, для локального запуска без контейнера нужно прописать local
short_url_length: 6 #длина сокращенных ссылок
paths: #настройки фиксированных путей
  logDir: ./log #папка с логами
  logName: shortner.log #файл с логом
  migrations: #настройка миграций
    dir: "migrations" #папка с миграциями
    recreate: false #обновлять стуктуру бд новыми миграциями
    drop: false #удалять таблицы при пересоздании
server: #настройки сервера
  timeout: 30s #тайм аут сервера
```

## Установка
Скачиваем репозиторий:

```bash
git clone https://github.com/CollapsorScripts/shortner.git
```

Заходим в папку проекта:

```bash
cd shortner
```

И запускаем команду

```bash
docker compose up -d
```

Командой в одну строку:
```bash
git clone https://github.com/CollapsorScripts/shortner.git && cd shortner && docker compose up -d
```

Развернется 2 контейнера:
**service** - контейнер с api gateway
**db** - контейнер с базой данных

## Запуск
Доступно 2 варианта запуска **в контейнере Docker** и **локальный запуск**

### Запуск в контейнере
Выставляем необходимые [настройки](#настройка) и в папке проекта выполняем команду 
```bash
docker compose up -d
```

### Локальный запуск
Сервис запускается с аргументами, при локальном запуске можно указать локальную конфигурацию.

Находясь в папке проекта создаем локальную конфигурацию:
```bash
touch config/local.yaml
```
И указываем настройки локальной среды:
```yaml
env: local #локальное окружение
short_url_length: 6
database: #настройки базы данных
  host: 127.0.0.1 #локальный или глобальный IP адрес сервера с базой данных
  port: 3050 #порт который прослушивает база данных
  user: db_user #имя пользователя БД
  password: db_password #пароль БД
  name: db_name #имя БД
paths: #настройки фиксированных путей
  logDir: ./log #папка с логами
  logName: shortner.log #файл с логом
  migrations: #Настройки миграций
    dir: "internal/database/migrations" #Папка с миграциями
    recreate: false #обновлять стуктуру бд новыми миграциями
    drop: false #удалять таблицы при пересоздании
server: #настройки сервера
  port: 8010 #порт который прослушивает api gateway
  timeout: 30s #тайм аут ожидания
```

Следующим шагом собираем конфигурацию с использованием **make**, в корне проекта выполняем команду:
```bash
make
```
Или собираем вручную:
```bash
go build -o bin/shortner ./cmd/entrypoint
```

И запускаем:
```bash
./bin/shortner --config=config/local.yaml
```

## Утилиты
В проекте используется утилита **make**, в корне проекта можно обнаружить файл **Makefile**.
Техническое описание:
```bash
make - Компилирует проект в папку bin под именем shortner и запускает с аргументами --config=config/local.yaml
make build - Компилирует проект в папку bin под именем shortner
make run_local - Запускает бинарник с локальной конфигурацией --config=config/local.yaml
make run_prod - Запускает бинарник с продакшен конфигурацией --config=config/prod.yaml
make gen_sqlc - генерирует sqlc файлы (настройки sqlc в файле sqlc.yaml корне проекта)
make update - обновляет зависимости проекта
```

## API
### Эндпоинт генерации сокращенной ссылки
```
POST: /api/v1/shorten
```

Тело запроса:
```json
{
  "url": "URL сайта"
}
```

Тело ответа:
```json
{
  "short_url": "сокращенная ссылка (без домена)"
}
```

### Эндпоинт запроса статистики сокращенной ссылки
```
GET: /api/v1/stats/{сокращенная ссылка}
```

Тело ответа:
```json
{
	"original_url": "оригинальная ссылка",
	"clicks": "количество кликов",
	"created_at": "время создания",
	"last_accessed_at": "время последнего посещения ссылки"
}
```

### Пример
Отправляем ссылку на генерацию сокращенной ссылки:
```bash
curl -X POST http://localhost:8010/api/v1/shorten -H "Content-Type: application/json" -d '{"url":"https://example.com"}'
```

Получаем ответ:
```json
{
  "short_url": "1BjkIH"
}
```

Далее можем проверить статистику этой ссылки:
```bash
curl http://localhost:8010/api/v1/stats/1BjkIH
```

Получаем ответ:
```json
{
	"original_url": "https://example.com",
	"clicks": 7,
	"created_at": "21/04/2026 17:34",
	"last_accessed_at": "21/04/2026 21:55",
}
```

## Принятые решения
> В качесиве кода редиректа был выбран **301**, так как подразумевается что ссылка генерируется на постоянно доступный ресурс и сокращенная ссылка будет часто использоваться, **301** позволит перенести SEO и закэшировать ссылку, при повторном посещении тем же клиентом не будет необходимости выполнять повторный запрос к серверу

> Метод генерации сокращенных ссылок - генерируется случайная строка из алфавита upper case, lower case и цирф от 0 до 9
При генерации идет проверка существует ли уже такая ссылка (если при создании ссылки сервис БД отдал **nil**, значит ссылка существует и нужно сгенерировать другую). Количество символов в сокращенной ссылке управляется с помощью настройки в конфигруции ```config/prod.yaml``` параметр ```short_url_length``` , по умолчанию стоит значение 6.
